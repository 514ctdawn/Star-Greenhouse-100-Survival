# 🔧 修复本地预览 404 错误

## 问题原因

`dist/index.html` 中的路径是为 GitHub Pages 配置的（`/Star-Greenhouse-100-Survival/assets/...`），但在本地预览时需要使用根路径（`/assets/...`）。

## ✅ 解决方案

### 方法 1：使用本地预览脚本（推荐）

**双击 `本地预览.bat`**

这会：
1. 使用本地配置重新构建（base path = `/`）
2. 启动预览服务器
3. 在浏览器中打开

### 方法 2：使用更新的双击打开HTML.bat

**双击 `双击打开HTML.bat`**

现在已经更新为使用本地预览配置。

### 方法 3：手动命令

```bash
# 构建本地版本
npm run build:local

# 预览
vite preview --config vite.config.local.js
```

然后访问：`http://localhost:4173`

## 📝 两种构建版本的区别

### GitHub Pages 版本（用于部署）
- 配置文件：`vite.config.js`
- Base path：`/Star-Greenhouse-100-Survival/`
- 用于：部署到 GitHub Pages

### 本地预览版本（用于本地测试）
- 配置文件：`vite.config.local.js`
- Base path：`/`
- 用于：本地预览和测试

## 🎯 现在试试

1. **双击 `本地预览.bat`**
2. 等待构建完成
3. 浏览器会自动打开 `http://localhost:4173`
4. 应该不会再有 404 错误了！

---

**记住：**
- 本地预览：使用 `本地预览.bat` 或 `双击打开HTML.bat`
- 部署到 GitHub Pages：使用 `npm run deploy`
