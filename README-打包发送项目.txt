  如何打包项目并发送给其他人
═══════════════════════════════════════════════════════

为什么 ZIP 文件可以打开？
─────────────────────────────────────────────────────

当项目被打包成 ZIP 并解压后，接收者可以：

1. 使用便携版（推荐）
   - 双击 "启动游戏.bat"
   - 自动选择可用的服务器（Python/Node.js）
   - 游戏在浏览器中打开

2. 使用开发服务器
   - 运行：npm install
   - 运行：npm run dev
   - 访问：http://localhost:5173

3. 使用预览服务器
   - 运行：npm install
   - 运行：npm run build:local
   - 运行：npm run preview:local
   - 访问：http://localhost:4173

═══════════════════════════════════════════════════════
创建便携版（推荐）
═══════════════════════════════════════════════════════

步骤：
1. 双击运行：创建便携版.bat
2. 等待创建完成
3. 将 Star-Greenhouse-Portable 文件夹压缩成 ZIP
4. 发送 ZIP 文件给其他人

接收者需要：
- Windows 系统
- Python 或 Node.js（可选，脚本会自动检测）

接收者操作：
1. 解压 ZIP 文件
2. 双击 "启动游戏.bat"
3. 游戏自动在浏览器中打开

═══════════════════════════════════════════════════════
打包整个项目
═══════════════════════════════════════════════════════

如果要发送完整项目（包含源代码）：

1. 压缩整个项目文件夹
2. 发送 ZIP 文件

接收者需要：
- Node.js 已安装
- 运行：npm install
- 运行：npm run dev

═══════════════════════════════════════════════════════
最小化版本（仅构建文件）
═══════════════════════════════════════════════════════

如果只想发送构建后的文件：

1. 运行：npm run build:local
2. 压缩 dist 文件夹
3. 添加一个启动脚本（见下方）

启动脚本内容：
─────────────────────────────────────────────────────
@echo off
cd dist
python -m http.server 8000
─────────────────────────────────────────────────────

接收者：
1. 解压文件
2. 双击启动脚本
3. 访问：http://localhost:8000

═══════════════════════════════════════════════════════
为什么不能直接打开 HTML？
═══════════════════════════════════════════════════════

直接打开 HTML 文件：
✗ 浏览器使用 file:// 协议
✗ 安全策略阻止加载 ES6 模块
✗ 出现 CORS 错误

使用 HTTP 服务器：
✓ 提供 HTTP 协议
✓ 可以正常加载模块
✓ 游戏正常运行

═══════════════════════════════════════════════════════
推荐方案
═══════════════════════════════════════════════════════

最佳方案：创建便携版
1. 运行：创建便携版.bat
2. 压缩 Star-Greenhouse-Portable 文件夹
3. 发送 ZIP 文件

优点：
✅ 包含所有必要文件
✅ 自动检测可用服务器
✅ 一键启动
✅ 不需要安装 Node.js（如果使用 Python）
